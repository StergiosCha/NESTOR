=== COQ SETUP ===

Declare a single base type for individuals and predicates as functions:

  Parameter Entity : Type.
  (* 1-place predicates for properties *)
  Parameter man woman student : Entity -> Prop.
  (* 2-place predicates for relations *)
  Parameter love read : Entity -> Entity -> Prop.
  (* Named individuals *)
  Parameter john mary : Entity.

This is standard Montague-style semantics in Coq. Entity is the domain
of individuals. Predicates map entities to propositions. This is NOT
type theory (MTT) where nouns would be separate types.

=== QUANTIFIER PATTERNS ===

"every man walks"       forall x : Entity, man x -> walk x
"some man walks"        exists x : Entity, man x /\ walk x
"no man walks"          ~ exists x : Entity, man x /\ walk x
                        OR equivalently: forall x : Entity, man x -> ~ walk x
"at least two men walk" exists x : Entity, exists y : Entity,
                          man x /\ man y /\ walk x /\ walk y /\ x <> y
"the man walks"         exists x : Entity, man x /\
                          (forall y : Entity, man y -> y = x) /\ walk x

Connectives: /\ (and), \/ (or), ~ (not), -> (implies), <-> (iff)

Adjective + noun: use conjunction.
  "tall man"  ->  man x /\ tall x      (NOT a separate type)

Relative clauses: conjoin inside the restriction.
  "every man who runs is tired"
  ->  forall x : Entity, (man x /\ run x) -> tired x

=== HOW TO DETERMINE THE RELATION ===

You must attempt TWO proofs. This is how we decide the relation:

STEP 1: Try to prove ENTAILMENT (premise implies hypothesis).
  Theorem entailment : (* formalised premise *) -> (* formalised hypothesis *).
  Proof. (* attempt *) Qed.

  If this proof goes through (Qed succeeds): the relation is ENTAILMENT.

STEP 2: Try to prove CONTRADICTION (premise implies negation of hypothesis).
  Theorem contradiction : (* formalised premise *) -> ~ (* formalised hypothesis *).
  Proof. (* attempt *) Qed.

  If this proof goes through (Qed succeeds): the relation is CONTRADICTION.

STEP 3: If NEITHER proof goes through, the relation is NEUTRAL/UNKNOWN.
  In this case, write both theorems but use Abort on both:

  Theorem entailment : (* ... *).
  Proof. (* stuck *) Abort.

  Theorem contradiction : (* ... *).
  Proof. (* stuck *) Abort.

IMPORTANT: always attempt both directions. Do not skip to Abort without trying.

=== PROOF TACTICS ===

Common tactics that work for NLI proofs:
  intro / intros    introduce hypotheses
  apply H           apply a hypothesis or lemma
  exact H           finish with an exact hypothesis
  exists t          provide a witness for existential
  destruct H        break apart /\ or \/
  split             prove a conjunction (left then right)
  left / right      choose a disjunct
  unfold X          expand a definition
  firstorder        automatic solver (try this first, often works)
  auto              simpler automatic solver
  tauto             propositional tautology solver

Strategy: try "firstorder" first. If it works, done. If not, do it manually
with intro/destruct/exists/apply.

=== EXAMPLES ===

Example 1 (entailment, using Montague-style every/some):
  Premise: "Every man is mortal. Socrates is a man."
  Hypothesis: "Socrates is mortal."

  Parameter Entity : Type.
  Parameter man mortal : Entity -> Prop.
  Parameter socrates : Entity.

  Axiom p1 : forall x : Entity, man x -> mortal x.
  Axiom p2 : man socrates.

  Theorem entailment : mortal socrates.
  Proof. apply p1. exact p2. Qed.

  Theorem contradiction : ~ mortal socrates.
  Proof. Abort. (* Cannot prove: entailment holds, not contradiction *)

Example 2 (contradiction, "no" quantifier):
  Premise: "No dog is a cat."
  Hypothesis: "Some dog is a cat."

  Parameter Entity : Type.
  Parameter dog cat : Entity -> Prop.

  Axiom p1 : forall x : Entity, dog x -> ~ cat x.

  Theorem entailment : exists x : Entity, dog x /\ cat x.
  Proof. Abort. (* Cannot prove: no witness available *)

  Theorem contradiction : ~ exists x : Entity, dog x /\ cat x.
  Proof. intro H. destruct H as [x [Hd Hc]]. apply (p1 x Hd). exact Hc. Qed.

Example 3 (neutral):
  Premise: "John is a student."
  Hypothesis: "John is a good student."

  Parameter Entity : Type.
  Parameter student good : Entity -> Prop.
  Parameter john : Entity.

  Axiom p1 : student john.

  Theorem entailment : student john /\ good john.
  Proof. Abort. (* Cannot prove: we don't know if John is good *)

  Theorem contradiction : ~ (student john /\ good john).
  Proof. Abort. (* Cannot prove either: John might be good *)

Example 4 (multi-premise chain, entailment):
  Premise: "Every European is a person. Every Italian is a European."
  Hypothesis: "Every Italian is a person."

  Parameter Entity : Type.
  Parameter european person italian : Entity -> Prop.

  Axiom p1 : forall x : Entity, european x -> person x.
  Axiom p2 : forall x : Entity, italian x -> european x.

  Theorem entailment : forall x : Entity, italian x -> person x.
  Proof. intros x Hi. apply p1. apply p2. exact Hi. Qed.

  Theorem contradiction : ~ (forall x : Entity, italian x -> person x).
  Proof. Abort.
